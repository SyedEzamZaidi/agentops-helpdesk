## Architectural insights pushed to project-context

- Insight #13: ServiceNow Zurich Inbound OAuth Client Credentials Trap (Entity Profile required, hidden by default UI)
- Insight #14: OAuth client secret used in both Authorization Code and Client Credentials grants (different security roles)
- Insight #15: Power Platform connector defaults are runtime-only, not Test-tab — Test tab does not auto-apply defaults
- Insight #16: Power Apps connector query parameters can only originate from URL query strings during import
- Insight #17: Power Apps connector type inference can produce wrong types; explicit type override needed for Boolean-named string params
- Insight #18: ServiceNow `priority` is a calculated field from `impact` × `urgency` — direct POST/PATCH overrides ignored
- Insight #19: POST responses resolve reference fields differently from GET; `sysparm_display_value=true` gives partial resolution on POST
- Insight #20: Power Platform custom connectors route through `*.azure-apim.net` API gateway; Bearer token on wire is caller's Azure AD token, gateway swaps in OAuth token server-side
