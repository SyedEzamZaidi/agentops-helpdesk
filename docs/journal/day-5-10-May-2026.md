# Day 5 Journal — 10-May-2026

## What got done
- Designed ServiceNow Agent end-to-end through 5 architectural questions:
  - Scope: ticket CRUD + audit logging from other agents
  - Audit pattern: two-call (start + end, log intent before action)
  - Authority: Position 3 — user confirms, IT reviews
  - Identity: caller_id = end user, separate field tracks AI origin
  - Response shaping: curated 7-field summary, not full record dump
- Decided trusted subsystem auth model with three layers (prompt filter, service account role, Business Rules)
- Drafted full agent instructions, committed `agents/servicenow/instructions.md`
- Started ServiceNow OAuth app registration

- Microsoft's pre-built ServiceNow connector is Premium — fine during trial, future cost item documented
- Microsoft connector vs custom connector decision: Plan C (managed now, custom in Week 3)
- ServiceNow's Inbound Integration UI hides redirect URL field for Client Credentials; classic UI exposes it
