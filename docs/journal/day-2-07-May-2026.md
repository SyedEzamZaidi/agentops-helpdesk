## Architectural decision logged

**Environment variables for external dependencies** — starting Week 2 (when we add the first custom connector / ServiceNow integration), all external endpoints, credentials, and configuration values will be exposed as Power Platform environment variables, not hardcoded.

Rationale:
- Solutions become truly portable across environments without manual surgery on import
- Secrets reference Azure Key Vault rather than being baked into the Solution
- Environment-specific values (dev ServiceNow vs test ServiceNow) handled cleanly
- Disaster recovery and demo scenarios become realistic — anyone can import and configure the Solution for their tenant in minutes

Trade-offs accepted:
- Slightly more setup per environment (define EV values during import)
- One extra layer of abstraction in the Solution

Triage Agent today has no external dependencies, so no EVs required yet. First EVs introduced when ServiceNow connector is added in Week 2.
