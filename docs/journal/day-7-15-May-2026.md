## OAuth terminology fixed

- **Authorization Code grant** — user-context, browser login, returns code → token
- **Client Credentials grant** — service-account, no user, client_id + secret → token
- **PKCE** — security add-on to Authorization Code (verifier/challenge prevents code interception); still user-context
- **Certificate-based auth** — Client Credentials variant using cert-signed JWT instead of secret
